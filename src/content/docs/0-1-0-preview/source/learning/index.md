This folder contains reusable learning artifacts that pair with the main [Learning roadmap](../learning-roadmap.md).

These documents are intentionally lightweight and contributor-oriented:

- they help a contributor turn the roadmap into concrete weekly output
- they should stay aligned with the current repo and docs graph
- they are not replacements for the product-facing architecture or component guides

## Current starters

- [Week 1 architecture summary starter](week-1-architecture-summary.md)
- [Cephalon glossary starter](cephalon-glossary-starter.md)

## Dated learning-pass records

These are dated records of autonomous learning passes against the docs graph,
the current `.NET` preview wave, and external industry signal. They do not
introduce policy; they record what was checked, what signal was observed, and
which owner-home doc should record any eventual durable change.

- [May 2026 learning pass](may-2026-learning-pass.md)

## How to use this folder

1. Start from [Learning roadmap](../learning-roadmap.md).
2. Use the week-by-week plan to decide what to read and run.
3. Use the starter documents in this folder to capture the outputs the roadmap asks for.
4. When a starter becomes stale, update it together with the related docs it references.
