Assembly-level marker indicating that the assembly contains auto-discovered
behavior types. When present, the engine uses the source-generated registration
class instead of runtime reflection scanning, resulting in zero-reflection startup.

```csharp
[AttributeUsage(AttributeTargets.Assembly, AllowMultiple = false, Inherited = false)]
public sealed class ContainsBehaviorsAttribute : Attribute
```

## Remarks

<p>
This attribute is automatically emitted by the <code>Cephalon.Behaviors.SourceGen</code>
source generator when it discovers one or more <code>[AppBehavior]</code> classes in the assembly.
You do not need to add it manually.
</p>
<p>
The <xref href="Cephalon.Abstractions.Behaviors.ContainsBehaviorsAttribute.RegistrationType" data-throw-if-not-resolved="false"></xref> property points to the generated class that provides
compile-time registration and topology descriptors, enabling the engine to skip
the expensive <code>DefinedTypes</code> / <code>GetCustomAttribute</code> reflection scan.
</p>
