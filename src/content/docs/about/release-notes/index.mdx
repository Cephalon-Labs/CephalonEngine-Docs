## Versioning policy

- **`0.1.0-preview.N`** — additive within the preview track. New behaviors, new packages, new schema sections land in `N+1` builds without breaking apps on `N`.
- **`0.2.0-preview` onward** — minor preview bumps signal **breaking schema or breaking API**. A migration page accompanies each one in **[Migration → Version upgrades](/0-1-0-preview/migration/version-upgrades/)**.
- **`1.0.0`** — the first stable release. Drops the `-preview` suffix. Locked surface: any change after this point follows SemVer (major = breaking, minor = additive, patch = bug-fix).
