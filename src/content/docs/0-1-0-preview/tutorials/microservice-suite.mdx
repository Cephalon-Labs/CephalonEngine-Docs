This tutorial takes the modular monolith from the **[First-app series](/0-1-0-preview/tutorials/first-app/)** and splits it across three hosts: a `Catalog` service, an `Orders` service, and an `Identity` service. By the end you'll have a Kubernetes deploy with shared eventing through Wolverine, OTLP traces that span the wire, and a shared identity provider.

## Skeleton recipes

- **Service-to-service auth** вҖ” wire Bearer JWT between hosts; see **[Reference вҶ’ Configuration вҶ’ Identity](/0-1-0-preview/reference/configuration/identity/)** + **[Technology вҶ’ Identity](/0-1-0-preview/technology/identity/)**. A dedicated *Identity tokens* tutorial lands in `0.2.0-preview`.
- **Cross-service eventing** вҖ” pattern in [Event-driven tutorial](/0-1-0-preview/tutorials/event-driven/).
- **Shared observability** вҖ” wire each service to the same OTLP backend; the `cephalon.engine.id` resource attribute keeps them separated.
- **Kubernetes deploy** вҖ” each host generates its own `deploy/kubernetes/` folder; deploy with `kubectl apply -k`.
