`Cephalon.EventSourcing` adds an event-sourced storage capability — appending to per-aggregate streams, replaying for projections, snapshot support. Adapter packages back the streams with a specific store.

<div class="ce-callout ce-callout--warn">
<span class="ce-callout__icon" aria-hidden="true">⚠️</span>
<div><strong>Maturity:</strong> the whole family is currently <code>M2</code>. Surface is settling; expect minor breaking schema changes within preview minors. <a href="/0-1-0-preview/reference/architecture/maturity/">Maturity audit</a>.</div>

## Install matrix

| Scenario | Packages |
| --- | --- |
| Event-sourced aggregate in EF Core | `Cephalon.EventSourcing` + `Cephalon.EventSourcing.EntityFramework` |
| Event-sourced + ClickHouse projection | + `Cephalon.EventSourcing.ClickHouse` |
| Multiple read-model targets | install one adapter per read-model target |

## Configuration shape

```jsonc title="appsettings.json"
{
  "Engine": {
    "EventSourcing": {
      "Enabled": true,
      "Provider": "EntityFramework",
      "Snapshots": { "Every": 100 },
      "Projections": {
        "RebuildOnSchemaChange": true
      }
    }
  }
}
```
